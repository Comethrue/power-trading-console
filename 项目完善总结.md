## 二、核心改进清单

### 1. 风险计算算法升级 ✅

**问题**：原算法使用虚假公式 `((trade_date.day * 11 + org_id * 3) % 100) / 100`，无业务意义。

**解决方案**：
- 基于近30天历史申报数据统计偏差率
- 偏差定义：申报价格偏离规则价格区间中点的程度
- 无历史数据时使用中性偏差率 0.25
- 偏差成本公式：`base_cost × (1 + deviation_ratio) × penalty_coef`

**代码变更**：
- 文件：`backend/app/services/risk_service.py`
- 新增函数：`_calc_deviation_ratio_from_history()`
- 行数：+60行

**测试覆盖**：
- `test_risk_query_no_history`：无历史数据场景
- `test_risk_query_with_history`：有历史数据场景

**解决方案**：

- 生成逐日明细统计
- 新增 API 端点：`GET /api/v1/settlement/reconcile/tasks/{task_id}/summary`

**代码变更**：

```sql

    contract_id   TEXT PRIMARY KEY,
```
